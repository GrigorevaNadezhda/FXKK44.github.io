4. Участники (actors) и цели (goals)

Участник Категория Цель
Владелец продукта Основной Представить интересы заказчика
Команда разработки Основной Разработать продукт (выполнить задачи)
Заказчик Внешний Предоставить требования к ПП
Система управления проектами Инструмент Предоставить КИС
Scrum-доска Инструмент Предоставить возможность отслеживать статусы задач

5. Предусловия (pre-conditions)

  • Заказчик сформировал свою идею и подготовил требования.
  • Система управления проектами установлена на ПК всех основных участников.
  • Основные участники ознакомлены с функционалом системы.

6. Постусловия (post-conditions)

  • ПП разработан в срок, указанный в договоре 
  • Ресурсы не превышены
  • Заказчик доволен ПП

7. Основной поток (main flow)

Участник Действие Ожидаемый резульат
Владелец продукта Собирает требования заказчика Составлен бэклог
Владелец продукта Участвует в оценке бэклога Задачи в бэклоге оценены по весу и приоритету
Владелец продукта Рекомендует приоритетные задачи в ходе планирования спринта Сначала выполняются самые важные задачи
Команда разработки Участвует в оценке бэклога Задачи в бэклоге оценены по весу и приоритету
Команда разработки Выполняет разработку продукта Задачи выполняются
Заказчик Предоставляет требования Владелец продукта получил требования и представление об их приоритете
Заказчик Оценивает выполненную работу Команда получает оценку качества ПП
